# Story E6.3: UI de Configuração de Distribuição na Aba Pesquisa

## Status
Done

## Story
**As a** administrador configurando estratégia de distribuição da pesquisa,
**I want** selecionar visualmente o modo de distribuição e receber feedback interativo sobre o impacto de cada escolha antes mesmo de salvar,
**so that** eu entenda a diferença entre "Aleatório" e "Rotativo" sem precisar ler documentação técnica — a própria UI me explica.

## Acceptance Criteria

1. - [x] Seção "Distribuição" da `ResearchTab.tsx` (criada em E2.2) exibe Radio Group estilizado com os 5 modos: Todas, Por Grupo, Individual, Aleatório, Rotativo.
2. - [x] Cada opção tem **tooltip explicativo** via shadcn `Tooltip` com 1 linha de texto simples:
      - Todas → "Eleitor vê e vota em todas as categorias da pesquisa"
      - Por Grupo → "Grupos configurados em cada campanha WhatsApp"
      - Individual → "Cada campanha apresenta uma única categoria"
      - Aleatório → "Sistema sorteia N categorias por eleitor (determinístico)"
      - Rotativo → "Distribuição equitativa em round-robin entre todos os eleitores"
3. - [x] **Modo "Aleatório"** selecionado → revela campo numérico: Slider + Input de "Máx. categorias por eleitor" (range: 1 a total de categorias ativas). Campo mapeado para `maxCategoriasPorEleitor`.
4. - [x] **Preview dinâmico** abaixo do seletor — compute e mostre em tempo real:
      - "Com **{N} categorias** ativas e modo **{modo}**, cada eleitor verá **~{X} categorias**"
      - N = contagem de segmentos da enquete (pode ser estimativa estática ou fetch lazy).
      - X calculado: Todas→N, Grupo→"depende da campanha", Individual→1, Aleatório→`maxCategorias`, Rotativo→"equitativo".
5. - [x] Animação suave (`animate-in fade-in slide-in-from-top-2 duration-200`) ao mostrar/ocultar campo de `maxCategorias`.
6. - [x] Valor padrão do modo definido conforme preset do tipo de pesquisa (vindo do E2.3): Premiação → "Por Grupo", Política/Pol. Públicas/Corporativo → "Todas".
7. - [x] Salva `modoDistribuicao` e `maxCategoriasPorEleitor` no form state (react-hook-form) e persiste via API da enquete.

```tsx

```
