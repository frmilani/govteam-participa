# Story E4.2: Componente DynamicResearchEngine

## Status
Done

## Story
**As a** administrador das instâncias satélites montando campanhas no Hub,
**I want** um componente visual que converta automaticamente os "modos" da minha enquete (NPS, Top of Mind, Listas, Avaliação) em etapas de navegação amigáveis para o eleitor,
**so that** um eleitor tenha fluxos polidos e lógicos sem que eu precise codar telas customizadas por tipo de eleição.

## Acceptance Criteria

1. - [x] Construir o React Component `<DynamicResearchEngine />` no escopo client (`"use client"`). Ele receberá via props `enqueteId`.
2. - [x] Ao ser montado na tela (useEffect ou React Query), o componente consome a Action/Fetch para a config vinda do E4.1.
3. - [x] A renderização condicional baseada na resposta da E4.1:
      - Se `modoDistribuicao == "todas"`, renderiza uma listagem em batch ou scroll único.
      - Se `modoDistribuicao == "grupo"`, renderiza paginação Wizard (Step 1, Step 2...).
4. - [x] Renderização dos Sub-Blocos da Categoria baseada no `modoColeta`:
      - "Top of Mind Puro" -> Renderiza um input HTML simples (Neste momento, blindado, sem bind lógico, apenas visual UI).
      - "Recall Assistido" -> Renderiza componente AutoComplete visualmente.
      - "Lista Sugerida" -> Box simples de radio buttons verticais/Grid baseado nas entidades que buscará depois.
5. - [x] Se o Bloco Categoria possuir um `TemplateQualidade` atrelado no payload da e4.1, o componente injetará abaixo da zona de voto de escolha (quando o usuário escolhe a entidade, o bloco da qualidade se revela).
6. - [x] UI: Handling de Loading principal da engine (Skeleton UI) enquanto o JSON de config viaja.
7. - [x] Responsável pela "State Machine" das respostas (Salvar objeto imenso localmente via Zustand ou Context API) antes do disparo de submissão final.
