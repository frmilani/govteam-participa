# Story E4.5: Sala de Consolidação (Top of Mind)

## Status
Done

## Story
**As a** administrador da pesquisa analisando o encerramento do pilar Top of Mind puro,
**I want** ver todos os votos em forma de lista ou texto que as pessoas escreveram aglomerados usando inteligência fonética,
**so that** não precise olhar a unha 5.000 votos isolados, eu apenas seleciono o aglomerado dominante (Padaria do ze 49 votos, Padaria seu zé 10 votos, BAdaria do Zeh 2vts) e transformo tudo na entidade "Padaria do Zé".

## Acceptance Criteria

1. - [x] Desenvolver a API de agrupamento inteligente: `GET /api/admin/enquetes/[id]/top-of-mind/grupos`.
2. - [x] O Backend puxará todos os "VotosLivres" da base com base no EnqueteID. Fazer a matemática de GroupBy nas tags/string de metaphone fonética que os unifica em memória e devolver as listas "Clusters" rankeados volume decrescente (Os mais polêmicos / errados primeiro).
3. - [x] Desenvolver Componente UI visual focado no fluxo (Sala de consolidação Tab / Page separada) mapeando a resposta e exibindo blocos flexões (Accordions informando Textos Originais vs Sugestões de Agrupamento).
4. - [x] Componente Backend POST e Front UI de Resolução: O gestor clica em "Consolidar Aglomerado para => NomeEntidade". O Backend reescreve a amarração dos VotosLivres (Tabelado no E1 - campo `consolidadoEmId`) fazendo referenciar formalmente um Elemento UUID `Estabelecimento`, permitindo a conversão do Top Of Mind cru para Analytics validável (Público nas pontas finais da aplicação analítica).
5. - [x] Ação de Criação Auxiliar: Se o aglomerado fonético é dominante e não existe a entidade para resolver, botão de UI do Action Group permitirá criar a própria Entidade nativamente extraindo o texto preferencial e logo injetando a amarra da AC4 nele. 
6. - [x] O Sistema deve omitir "clusters já consolidados" nas lógicas de refresh.
