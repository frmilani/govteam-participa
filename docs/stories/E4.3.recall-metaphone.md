# Story E4.3: Recall Assistido com Metaphone PT-BR

## Status
Done

## Story
**As a** cidadão digitando a entidade da minha escolha no modo Recall Assistido,
**I want** que o motor liste alternativas prováveis da base da cidade mesmo se eu errar sílabas ou letras (ex: 'Farmácia' vs 'Formácia', 'Zé' vs 'Ze'),
**so that** meu voto caia diretamente na urna contabilizada sem gerar estresse e lixo desnecessário para o admin consolidar depois.

## Acceptance Criteria

1. - [x] O componente visual `AutocompleteRecall.tsx` no front-end realiza requisições debounceadas ao backend pesquisando o que é digitado no input.
2. - [x] Utilidade backend (Lib/Service) importa ou cria um algorítmo Metaphone focado na língua Portuguesa Brasileira (`metaphone-ptbr-node` module ou equivalente/portado manual se preciso).
3. - [x] Endpoint do Autocomplete (Backend API): Rota exclusiva `/api/public/enquetes/autocomplete/[categoriaId]` recebe string textual e procura a entidade.
4. - [x] Regra de Fuzzy do Endpoint (PRD): 
      - Regra 1: Match string direto `contains` nativo no Prisma. (Rank máximo)
      - Regra 2: Converte query input em Chave Fonética Metaphone. Analisa a base em tempo real e acha entidades cuja chave fonética bata. (Rank Médio)
      - O payload retorna no MÁXIMO 5 sugestões para o frontend renderizar no dropdown de apoio.
5. - [x] Frontend UX: Se o usuário ignorar as 5 dicas e prosseguir, seu texto puro é mantido no state para submissão posterior (`allowCustomValue=true`), caracterizando o input como fallback de VotoLivre.
6. - [x] Indexação Nativa (Worker ou Trigger no prisma model, ou on Create): Quando cria-se a entidade (`Estabelecimento`) o campo invisível/auxiliar de chaves metaphone deve ser pre-calculado e gravado nela para otimizar busca.
