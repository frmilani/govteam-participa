# Story E6.1: Motor de Distribuição de Categorias

## Status
Done

## Story
**As a** motor de pesquisa (DynamicResearchEngine) servindo milhares de eleitores com campanhas distintas,
**I want** receber a informação de qual campanha originou o acesso e automaticamente apresentar ao eleitor somente as categorias relevantes para aquele segmento de campanha,
**so that** um lead recrutado pela campanha "Padaria e Confeitaria" não precise avaliar as "Clínicas Odontológicas" — reduzindo abandono e aumentando a qualidade dos dados coletados.

## Acceptance Criteria

1. - [x] Endpoint `GET /api/public/enquetes/[id]/research-config` (E4.1) extendido para aceitar query param `?campanhaId=xxx` ou `?trackingHash=yyy`.
2. - [x] **Modo "Todas"**: retorna todas as categorias ativas. Sem filtro. (Caso default sem parâmetro de campanha.)
3. - [x] **Modo "Por Grupo"**: filtra as categorias pelo(s) `grupoIds[]` vinculados à campanha identificada. Se campanha não tem grupos, retorna todas (fallback seguro).
4. - [x] **Modo "Individual"**: retorna exatamente 1 categoria vinculada à campanha. Erro 400 se campanha vínculo não definiu categoriaId.
5. - [x] **Modo "Aleatório"**: sorteia N categorias (N = `maxCategoriasPorEleitor`) de forma **determinística por lead** — mesmo `leadId + enqueteId` sempre sorteia as mesmas categorias. Seed: `crc32(leadId + enqueteId)`. Se não houver `leadId` (anônimo), seed aleatório por acesso.
6. - [x] **Modo "Rotativo"**: distribui categorias em round-robin entre acessos. Contagem de acessos por categoria persistida em tabela `DistribuicaoRotativa { enqueteId, categoriaId, totalAcessos }`. Incrementa atomicamente via Prisma `upsert`.
7. - [x] Se acesso sem campanha (`campanhaId` ausente) → usar `modoDistribuicao` padrão da enquete.
8. - [x] Logging: `[DISTRIBUICAO] modo=aleatorio enqueteId=X categorias=[A,B,C] leadId=Y`.
9. - [x] Testes unitários para cada modo (especialmente determinismo do aleatório e equilíbrio do rotativo).

## Dev Notes

### Algoritmo de seed determinístico simples
```typescript
function mulberry32(seed: number) {
  return () => {
    seed |= 0; seed = seed + 0x6D2B79F5 | 0;
    let t = Math.imul(seed ^ seed >>> 15, 1 | seed);
    t = t + Math.imul(t ^ t >>> 7, 61 | t) ^ t;
    return ((t ^ t >>> 14) >>> 0) / 4294967296;
  };
}
const seed = hashCode(leadId + enqueteId); // hashCode = simples CRC32 in JS
```
