# Story E2.1: Campo "Tipo de Pesquisa" + Visibilidade Condicional de Abas

## Status
Done

## Story
**As a** administrador configurando uma enquete,
**I want** selecionar o "Tipo de Pesquisa" na aba Dados Básicos e ver o formulário se adaptar inteligente ocultando/mostrando abas,
**so that** eu tenha uma experiência focada apenas nas configurações relevantes para o meu caso de uso atual.

## Acceptance Criteria

1. - [x] Adicionar campo `tipoPesquisa` (string, enum implícito) ao schema Zod do `EnqueteForm`.
2. - [x] Criar seletor visual na aba "Dados Básicos" (`BasicTab.tsx`) com os tipos: 🏆 Premiação, 🗳️ Política, 🏛️ Pol. Públicas, 💼 Corporativo, 🌟 Custom.
3. - [x] O `tipoPesquisa` deve persistir no banco de dados (schema Prisma já preparado no Epic E1).
4. - [x] Aba "Categorias" deve alterar o label visual internamente dependendo do `tipoPesquisa`: "Cargos Eletivos" (Política), "Áreas Temáticas" (Pol. Pub.), "Dimensões" (Corp.). O ID interno pode continuar `segments`.
5. - [x] Aba "Entidades" (tratado no E1.3) já usa lógica parecida, mas aqui conectamos isso ao `watch('tipoPesquisa')` formalizado.
6. - [x] Abas que não são relevantes para o tipo (ex: Entidades em Top of Mind puro, que será setado pelo preset na próxima story) podem ficar "collapsed" (opacidade reduzida, clicáveis para override).
7. - [x] Aba "🎁 Premiação" fica visível em TODOS os tipos, mas:
   - Premiação → toggle interno (da aba) ON por padrão.
   - Demais → toggle interno OFF por padrão (ativável).
8. - [x] Tipo "Custom" desbloqueia todas as abas sem predefinições restritivas.
9. - [x] Default para novas enquetes é `premiacao` (backward compatible).
