# Story E6.2: Segmentação Avançada nas Campanhas WhatsApp

## Status
Done

## Story
**As a** gestor de campanha configurando o disparo de WhatsApp para 5.000 leads,
**I want** dividir o universo de disparo em grupos de categorias com pesos percentuais definidos (ex: 60% recebem link focado em "Alimentação" e 40% em "Saúde"),
**so that** cada eleitor seja convidado para avaliar a categoria mais relevante para seu perfil, aumentando a taxa de conclusão da pesquisa.

## Acceptance Criteria

1. - [x] Modelo `Campanha` (existente, RF-007) estendido com campo `segmentacao Json?` no schema Prisma. Backward compatible — campo opcional com default null.
2. - [x] Migration aplicada sem breaking changes nas campanha existentes.
3. - [x] API `PUT /api/admin/campanhas/[id]` aceita e persiste o novo campo `segmentacao` no body.
4. - [x] Estrutura do JSON de segmentação validada por Zod:
    ```typescript
    const segmentacaoSchema = z.object({
      tipo: z.enum(['todos', 'grupoCategoria', 'individual']),
      grupoIds: z.array(z.string().cuid()).optional(),
      pesoDistribuicao: z.record(z.string(), z.number().min(0).max(100)).optional(),
    }).refine(data => {
      if (data.pesoDistribuicao) {
        const total = Object.values(data.pesoDistribuicao).reduce((a, b) => a + b, 0);
        return total === 100;
      }
      return true;
    }, { message: "Soma dos pesos deve ser 100%" });
    ```
5. - [x] **Wizard de Campanha (Step 2)** extendido com uma nova seção colapsável "Segmentação por Categorias":
      - Radio: "Todos os leads recebem o mesmo link" (default) vs "Segmentar por grupo de categorias".
      - Se "Segmentar": checklist com segmentos/grupos da enquete vinculada.
      - Se múltiplos grupos selecionados: input de peso (%) por grupo com validação de soma = 100%.
6. - [x] **Preview de Segmentação (Step 5 — Revisão)**: exibir quais categorias cada grupo inclui, quantos leads estimados por grupo.
7. - [x] O tracking link gerado para cada lead é acrescido de `?campanhaId={id}` automaticamente (já pode existir, confirmar comportamento no trigger de disparo BullMQ).
8. - [x] Campanhas existentes sem `segmentacao` → comportamento idêntico ao atual (sem filtro de grupo).
