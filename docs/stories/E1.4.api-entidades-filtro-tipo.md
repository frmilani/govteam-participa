# Story E1.4: API de Entidades — Filtro por Tipo + Metadados

## Status
Done

## Story
**As a** desenvolvedor consumindo a API de estabelecimentos,
**I want** poder filtrar entidades por tipo e persistir metadados específicos do tipo,
**so that** o frontend consiga exibir apenas candidatos para pesquisa política, apenas empresas para premiação, etc.

## Acceptance Criteria

1. - [x] `GET /api/estabelecimentos?tipo=CANDIDATO` filtra por tipo
2. - [x] `GET /api/estabelecimentos` sem filtro retorna todos (backward compatible)
3. - [x] `POST /api/estabelecimentos` aceita campos `tipo` e `metadados` no body
4. - [x] `PUT /api/estabelecimentos/[id]` aceita atualização de `tipo` e `metadados`
5. - [x] Validação: `tipo` deve ser valor válido do enum `TipoEntidade` (ou omitido para default EMPRESA)
6. - [x] Validação: `metadados` deve ser JSON válido quando presente
7. - [x] Resposta da API inclui `tipo` e `metadados` nos objetos retornados
8. - [x] Request público (`GET` sem auth com `organizationId` param) também retorna `tipo` e `metadados`
9. - [x] Hub auth (spoke-to-spoke) também suporta filtro por tipo

**IMPORTANTE:** O filtro `tipo` deve funcionar nos 3 caminhos. O filtro é aplicado APÓS a autenticação (L98-104) na chamada ao service.
