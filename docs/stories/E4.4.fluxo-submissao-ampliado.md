# Story E4.4: Fluxo de Submissão Ampliado (POST /api/submissions)

## Status
Done

## Story
**As a** engine de frontend coletando dados do eleitor,
**I want** submeter um pacote final maciço contendo dezenas de respostas simultâneas em diversos modelos (votos convencionais, votos livres cegos, questionários de qualidade e lógicas adicionais),
**so that** o Backend faça o parser e grave de modo seguro transacional essas informações sem corromper parte da urna.

## Acceptance Criteria

1. - [x] Ampliar rota/serviço `POST /api/public/submissions`.
2. - [x] O payload JSON submetido (novo contrato da E4) precisará agrupar lógicas diversificadas:
      - `VotoEstabelecimento` atrelados na Category X (Voto de lista, comum já existente na V1 brownfield).
      - `VotoLivre` na Category Y (O Top Of Mind "Free Text", com trigger do metaphone keys pra salvar sua chave nele).
      - `RespostaQualidade` vinculada seja ao voto comum seja ao voto livre, apontando para a PK das `PerguntaQualidade` e da enquete (respostaId base).
3. - [x] Garantir o isolamento lógico das enquetes (urnas blindadas). Bloqueio automático 403 se "Status da Enquete !== EM_ANDAMENTO" no ato do fetch final.
4. - [x] Todas inserções devem ocorrer atreladas usando arquitetura Prisma Nested Inserts via `$transaction([])` englobando 1 Resposta Global (Ticket / Cookie Fingerprint base) com sub-votos múltiplos conectados a ele (Vínculo nativo).
5. - [x] Validação rigorosa: Não permita que o JSON envie respostas com `categoriaId` não pertencentes à Enquete atual consultada na hora transacional, evitando injection lateral.

## Dev Notes

Este é o filete central de segurança da urna. Na arquitetura PRD, cita-se o Termo "Fire-and-forget telemetria para Hub (SEM dados sensíveis, CA-G1)". Pondere usar `Promise.resolve` fora da via crítica após finalizar o voto do eleitor e fechar o request, engatilhando por microservico HTTP pra Hub só um Count++ confirmando uma submissão de eleitor, ajudando gerência Hub sem precisar expor as chaves qualitativas (isolamento Spoke garante zero vazamentos pro SaaS).
