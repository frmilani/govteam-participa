# Story E2.3: Presets Automáticos por Tipo de Pesquisa

## Status
Done

## Story
**As a** administrador inexperiente do sistema de pesquisas,
**I want** que o motor inicialize as opções complexas sozinho ao escolher um Tipo de Pesquisa,
**so that** eu não precise configurar manualmente os modos de coleta ideais, distribuições e configurações de prêmios se eu só quero um formato padrão.

## Acceptance Criteria

1. - [x] Lógica para quando o usuário alterar `tipoPesquisa` em Dados Básicos, disparar um aviso (Toast ou Dialog) e aplicar presets automáticos se aprovado (para não destruir config manual sem dolo).
2. - [x] Preset "Premiação Empresarial": Modo=Recall Assistido, Qualidade=ON, Distribuição=Por Grupo, Premiação=ON, Blocos=OFF.
3. - [x] Preset "Pesquisa Política": Modo=Top of Mind Puro, Qualidade=OFF, Distribuição=Todas, Premiação=OFF, Blocos=OFF.
4. - [x] Preset "Políticas Públicas": Modo=Priorização, Qualidade=ON, Distribuição=Todas, Premiação=OFF, Blocos: NPS(visível), Texto Livre(visível).
5. - [x] Preset "Corporativo": Modo=Recall Assistido, Qualidade=ON, Distribuição=Todas, Premiação=OFF, Blocos: NPS(visível), Texto Livre(visível).
6. - [x] Preset "Custom": Nenhum hook forçado, todas opções desbloqueadas.
7. - [x] O usuário pode editar depois, os presets apenas geram *default values* ou mutações na seleção se ele confirmar.
8. - [x] Garantir idempotência: O sistema só oferece aplicar o preset se for uma troca manual de tipoPequisa feita pelo UI pós-load; na inicialização (fetch load) NÃO re-aplicar e sobrescrever o estado vindo do banco.

## Dev Notes

### Cuidado crítico: Lifecycle de React Hook Form
Não confie no `watch('tipoPesquisa')` dentro de um useEffect para disparar essas substituições de preset, senao os defaults vindos da API podem ser sobrescritos acidentalmente toda vez que carregar a tela.
Faça o binding do evento atrelado estritamente à Ação do Usuário. No Grid de Tipos (BasicTab), no evento `onClick`, altere o tipo e imediatamente set os outros form fields.

### Testing
- Abrir form de EDIÇÃO: Conferir que carregou do backend, valores não mudaram via javascript local.
- Alterar aba Tipo para outra: Conferir visual e valores (NPS ligou? Premiação desabilitou?).
