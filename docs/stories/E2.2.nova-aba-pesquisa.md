## Status

## Story
**As a** administrador,
**I want** uma aba dedicada a "Pesquisa" no formulário de Enquete com todas as opções de configuração do motor,
**so that** eu consiga definir o modo de coleta de votos, critérios de validação (NPS, qualidade, etc.) e regras de distribuição.

## Acceptance Criteria

1. - [x] Criar novo componente `ResearchTab.tsx` para abrigar as configurações.
2. - [x] Inserir aba "🔬 Pesquisa" no `EnqueteForm` na posição: `[Dados Básicos] → [Categorias] → [🔬 Pesquisa] → [Entidades]...`
3. - [x] Adicionar os campos persistentes ao Schema Zod (`modoColeta`, `incluirQualidade`, `modoDistribuicao`, `maxCategoriasPorEleitor`, `randomizarOpcoes`).
4. - [x] Criar Seção **Modo de Coleta** com opções (Radio Group estilizado): "Top of Mind Puro", "Recall Assistido", "Lista Sugerida", "Recall Duplo".
5. - [x] Adicionar checkbox extra de configuração "Permitir resposta 'Não sei/Não conheço'" associado ao formato de validação (parte de `configPesquisa`).
6. - [x] Criar Seção **Qualidade** com Toggle Switch "Incluir perguntas de qualidade".

10. - [x] A opção "Recall Duplo" aparece somente se tipoPesquisa for Política ou Corporativo.
11. - [x] Todos os novos campos se comunicam com os dados vindos via Prisma model (`Enquete`).
