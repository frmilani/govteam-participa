# Story E2.4: Regras de Interação UI da Aba Pesquisa

## Status
Done

## Story
**As a** administrador preenchendo as configurações,
**I want** ver e interagir de forma reativa com o formulário expandindo seções secundárias apenas quando ativos as engines primárias,
**so that** não fique um formulário maciço e assustador parecendo um avião comercial.

## Acceptance Criteria

1. - [x] Interação: Ao selecionar Modo="Top of Mind Puro", toda uma mensagem/alerta "Neste modo, o motor não apresentará opções. Cadastros prévios não são exibidos." deve surgir perto do controle. E aba Entidades é visualmente definida como collapsed em UI.
2. - [x] Interação: Ao habilitar "Recall Duplo", expandir opções informativas "Irá criar passo 1 puro, passo 2 assistido".
3. - [x] Interação: Ao desabilitar "Qualidade", deve colapsar/desaparecer todo input extra sobre templates (que faremos na fase próxima do E3).
4. - [x] Interação: Selecionar "Aleatório" em tipo de distribuição libera campo "Máx categorias" e "Mín Categorias".
5. - [x] Interação: Toggles da seção de blocos adicionais (NPS, etc) podem expandir settings contextuais (mesmo que estáticas de aviso no momento).
6. - [x] Uso de Animações CSS: Componentes expand/collapse usam Tailwind `animate-in fade-in` ou bibliotecas framer/headlessui compatíveis com o estado do sistema (collapsible pattern do shadcn/ui funciona bem).
