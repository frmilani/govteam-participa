# Story E1.2: UI Adaptativa — CRUD de Entidades

## Status
Done

## Story
**As a** administrador do sistema,
**I want** o formulário de criação/edição de entidades (estabelecimentos) se adaptar ao tipo selecionado (Empresa, Candidato, Proposta, etc.),
**so that** eu consiga cadastrar entidades de qualquer tipo com os campos relevantes sem confusão.

## Acceptance Criteria

1. - [x] Seletor de `TipoEntidade` no formulário de criação/edição (dropdown ou radio group)
2. - [x] Tipo `EMPRESA` → exibe campos de contato: telefone, whatsapp, instagram, facebook, website, endereco
3. - [x] Tipo `CANDIDATO` → exibe campos em `metadados`: partido, numeroCandidato, cargo, coligacao
4. - [x] Tipo `PROPOSTA` → exibe campos em `metadados`: areaOrcamentaria, impactoEstimado, prazoExecucao
5. - [x] Tipo `SERVICO_PUBLICO` → exibe campos em `metadados`: secretaria, localAtendimento
6. - [x] Tipo `DIMENSAO` → exibe campos em `metadados`: areaCorporativa, peso
7. - [x] Tipo `MARCA` → exibe campos em `metadados`: segmentoMercado, paisOrigem
8. - [x] Tipo `OUTRO` → exibe editor JSON livre para metadados
9. - [x] Filtro por tipo na listagem de entidades
10. - [x] Campos de empresa (endereco, tel, etc.) ficam ocultos quando tipo NÃO é EMPRESA
11. - [x] Dados existentes (tipo EMPRESA) continuam funcionando normalmente
12. - [x] Formulário envia `tipo` e `metadados` no payload para a API
