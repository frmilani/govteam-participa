# Story E5.2: APIs de Analytics Demográfico

## Status
Done

## Story
**As a** administrador responsável por entregar relatórios ao cliente pagante,
**I want** 4 APIs de analytics prontas que agreguem votos, qualidade e dados demográficos em formatos consumíveis,
**so that** o Dashboard do Admin mostre gráficos ricos sem eu precisar extrair CSVs manualmente nem violar a privacidade dos eleitores.

## Acceptance Criteria

1. - [x] `GET /api/admin/enquetes/[id]/analytics/overview` — retorna: total respostas, taxa de conclusão (respostas / eleitores notificados), período de votação, split de tipo de voto (lista, livre, recall).
2. - [x] `GET /api/admin/enquetes/[id]/analytics/ranking` — retorna ranking consolidado de entidades por votos (VotoEstabelecimento + VotosLivres consolidados), com: posição, nome entidade, total votos, percentual do total. Filtro query param `?categoriaId=`.
3. - [x] `GET /api/admin/enquetes/[id]/analytics/demographics` — retorna: para cada campo demográfico coletado, distribuição de eleitores (ex: `{bairro: {"Centro": 120, "Norte": 80}}`). Aceita filtro `?estabelecimentoId=` para cruzar "eleitores dessa entidade por bairro".
4. - [x] `GET /api/admin/enquetes/[id]/analytics/qualidade` — retorna: por categoria, média das RespostaQualidade por pergunta. Aceita filtro `?categoriaId=`. Retorna estrutura de Radar (pergunta + média).
5. - [x] **Regra Urna Trancada**: nenhuma API retorna dados identificáveis (quem votou em quem). Apenas agregações.
6. - [x] Threshold de anonimidade: grupos com menos de 5 respostas são omitidos do resultado com nota `"insuficiente": true`.
7. - [x] Todas as rotas protegidas por autenticação admin (`auth()` + `checkPermission`).
8. - [x] Cache de 5min via `revalidate` da rota Next.js ou usando `unstable_cache` para evitar queries pesadas a cada request.
9. - [x] Performance: responder em < 2s para enquetes com até 10.000 respostas.

## Dev Notes

### Exemplo de query GroupBy para ranking
```typescript
const ranking = await prisma.votoEstabelecimento.groupBy({
  by: ['estabelecimentoId'],
  where: { resposta: { enqueteId } },
  _count: { estabelecimentoId: true },
  orderBy: { _count: { estabelecimentoId: 'desc' } },
  take: 50,
});
```

### Regra dos 5+ votos (threshold)
```typescript
return grupos.filter(g => g._count >= 5);
```
