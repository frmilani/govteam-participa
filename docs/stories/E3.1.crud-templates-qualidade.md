# Story E3.1: CRUD de Templates de Qualidade

## Status
Done

## Story
**As a** administrador preocupado com a consistência metodológica,
**I want** criar e gerenciar Templates de Qualidade contendo perguntas de vários tipos (rating, likert, texto),
**so that** eu possa reutilizá-los em múltiplas categorias de pesquisa, padronizando a coleta de dados de qualidade do serviço das entidades que recebem recall da marca.

## Acceptance Criteria

1. - [x] `GET /api/admin/templates-qualidade` — lista os templates criados pela organização autenticada.
2. - [x] `POST /api/admin/templates-qualidade` — cria um template com nome e array aninhado (CreateNested) das perguntas.
3. - [x] `PUT /api/admin/templates-qualidade/[id]` — edita um template (modifica nome e atualiza as perguntas vinculadas, possivelmente apagando as velhas iterativamente ou usando sync tools do Prisma).
4. - [x] `DELETE /api/admin/templates-qualidade/[id]` — remove um template e faz Cascade nas perguntas que são dependentes exclusivamente dele (já feito no migration cascade model, validar on runtime).
5. - [x] Validação na persistência (Zod): Pelo menos 1 pergunta por template.
6. - [x] Validação na persistência (Zod): Array `opcoes` é obrigatório quando o `tipo` da pergunta for `likert` ou similar baseado em múltiplas escolhas limitadas definidas no PRD.
7. - [x] Multi-tenant: Isolamento mandatário por `organizationId` em todos CRUD operations no Serviço Backend.
8. - [x] UI do Administrador (Página e Modal): Tabelas ou cards listando o acervo de templates com ação de Editar/Deletar/Duplicar.
9. - [x] Formulário dinâmico UI (FieldArray) em `react-hook-form` lidando com o acréscimo de 'n' perguntas qualitativas (Drag-and-Drop / setOrder opcional para reordenar, input flexível para tipos diferentes).

## Dev Notes

### `updateTemplate` Behavior
Atualizar recursos aninhados no Prisma geralmente requer uma estratégia `deleteMany` (para limpar as relações velhas de `PerguntaQualidade`) antes de recriar as novas pelo payload unificado salvo através do frontend (só um `create` nas perguntas do update payload), devido à reordenação caótica de arrays pelo user. Certifique-se de envolver em um `$transaction`.

### Tipagem de Tipo Pergunta
Trate os "Tipos" definidos globalmente.
Ex do PRD: `RATING_5`, `RATING_10`, `LIKERT`, `SIM_NAO`, `TEXTO_LIVRE`.
