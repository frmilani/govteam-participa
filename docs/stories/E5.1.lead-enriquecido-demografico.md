# Story E5.1: Enriquecimento Automático do Lead com Dados Demográficos

## Status
Done

## Story
**As a** administrador querendo entender o perfil demográfico do eleitorado,
**I want** o sistema automaticamente capturar e salvar no Lead os dados extras preenchidos no formulário do Hub (bairro, faixa etária, profissão),
**so that** eu possa futuramente cruzar "Padarias mais votadas por bairro" sem acionar processo manual de exportação e VLOOKUP no Excel.

## Acceptance Criteria

1. - [x] No processamento da submissão (`POST /api/public/submissions`), após vincular o Lead à resposta, extrair do JSON recebido a seção de campos demográficos.
2. - [x] Os campos demográficos são identificados por uma whitelist configurável de chaves (ex: `["bairro", "faixaEtaria", "profissao", "escolaridade", "renda"]`) que não inclui campos PII diretos (`cpf`, `whatsapp`, `email`, `nome`).
3. - [x] Persistir os campos extraídos como objeto JSON no campo `dadosDemograficos` do Lead: ex: `{ "bairro": "Centro", "faixaEtaria": "25-34" }`.
4. - [x] Se o Lead já possui `dadosDemograficos`, fazer **merge** (não sobrescrever): novos campos são adicionados, valores existentes são mantidos.
5. - [x] Se a submissão não contém campos demográficos reconhecidos, `dadosDemograficos` permanece inalterado — sem erro, sem log ruidoso.
6. - [x] Nenhum campo PII sensível (CPF, whatsapp, nome completo, e-mail) deve ser gravado dentro de `dadosDemograficos` — a implementação deve ter filtro proativo.
7. - [x] Log de auditoria (nível INFO) listando apenas as **chaves** capturadas (nunca os valores): `[LEAD_ENRICH] leadId=xxx campos=["bairro","faixaEtaria"]`.

## Dev Notes

### JSON Merge no Prisma
O Prisma não faz merge automático de campos Json. Estratégia:
```typescript
const existing = lead.dadosDemograficos as Record<string, any> ?? {};
const merged = { ...existing, ...newDemographics };
await prisma.lead.update({ where: { id: leadId }, data: { dadosDemograficos: merged } });
```
